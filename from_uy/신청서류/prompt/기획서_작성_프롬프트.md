# Guide for Writing AI Model Development Proposal (English-to-Korean Pipeline)

This document provides instructions on how to use the English-written prompts in the `prompt/` directory to generate a high-quality Korean proposal.

## 📁 Prompt File Structure (`prompt/`)

1. **Phase 1: Initialize System**: [00_System_Prompt.md](file:///e:/muckja999/%EA%B3%B5%EB%AA%A8%EC%A0%84/%EC%A0%9C2%ED%9A%8C%20%E3%80%8C%EA%B3%B5%EC%A0%95%EC%9C%84%20AI%C2%B7%EB%8D%B0%EC%9D%B4%ED%84%B0%E3%80%8D%ED%99%9C%EC%9A%A9%20%EA%B3%B5%EB%AA%A8%EC%A0%84/%EC%8B%A0%EC%B2%AD%EC%84%9C%EB%A5%98/prompt/00_%EC%8B%9C%EC%8A%A4%ED%85%9C_%ED%94%84%EB%A1%AC%ED%94%84%ED%8A%B8.md)
   - Written in English for maximum logical performance.
   - Sets the AI's persona as an elite technical writer and initiates the Ask-Before-Answer (ABA) strategy.
   - Enforces strict formatting rules: No arrows, no parentheses, and noun-ending lists.

2. **Phase 2: Chapter-by-Chapter Generation**:
   - Provide the following technical input files when prompted by the AI:

   - **Chapter II**: [02_Ⅱ_Proposal_Details.md](file:///e:/muckja999/%EA%B3%B5%EB%AA%A8%EC%A0%84/%EC%A0%9C2%ED%9A%8C%20%E3%80%8C%EA%B3%B5%EC%A0%95%EC%9C%84%20AI%C2%B7%EB%8D%B0%EC%9D%B4%ED%84%B0%E3%80%8D%ED%99%9C%EC%9A%A9%20%EA%B3%B5%EB%AA%A8%EC%A0%84/%EC%8B%A0%EC%B2%AD%EC%84%9C%EB%A5%98/prompt/02_%E2%85%A1_%EA%B8%B0%ED%9A%8D%EC%84%9C_%EC%83%81%EC%84%B8%EB%82%B4%EC%9A%A9.md)

## 🚀 Execution Workflow

1. Copy the content of **File 00** and paste it into the AI.
2. The AI will acknowledge in Korean and ask questions for "Ⅰ. Proposal Overview."
3. Copy the content of **File 01** and paste it as the answer.
4. Review the generated Korean text and repeat the process for Files 02 and 03.
5. Compile the final Korean text into the `인공지능 모델 개발 기획서.hwpx` document.

---

## ⚖️ Critical Constraints (Mandatory)

1. **No Arrows (`->`)**: Always use numbered lists (1, 2, 3) for sequences or analysis steps.
2. **No Parentheses (`()`)**: Describe details in full, flowing sentences. (Exception: URLs in the Miscellaneous chapter).
3. **URL Documentation**: In the Miscellaneous chapter, explicitly include the URL link in parentheses for each data source and reference item.
4. **Data Source Selection**: Use only publicly available, internet-searchable data sources (e.g., official FTC websites), not internal logs or synthetic data.
5. **No Phonetic Korean for Tech Names**: Use original English (e.g., **BGE-M3**, not **비지이엠쓰리**).
6. **Noun Phrase Endings**: List items and references must end with noun phrases (e.g., **~구축**, **~실시**).
5. **Technical Accuracy**: Clearly state that **Sensitivity Analysis** is being conducted with multiple model candidates and that the final model will be an **8B or larger** parameter model.
