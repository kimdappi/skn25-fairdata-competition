# QA 데이터셋 생성 가이드라인

## 목적
RAG 파이프라인의 Recall 성능을 평가하기 위한 **질문(query) → 정답 chunk_id 5개** 쌍을 만드는 방법론

---

## 1. QA 쌍의 구조

```json
{
  "id": "Q-S-001 / Q-M-001",
  "query": "사람이 자연어로 물어볼 법한 질문",
  "source_doc": "의결서 제목 (복수 의결서의 경우 여러 개 나열)",
  "answer_chunks": [
    {
      "chunk_id": "chunk_id_1",
      "answer_reason": "각 chunk가 정답인 이유 (기업명과 위반사실 포함)"
    },
    ...
  ]
}
```

---

## 2. 질문(query) 작성 원칙

### 질문 유형 분류

| 유형 | 질문 패턴 | 정답이 위치하는 섹션 |
|---|---|---|
| **위반행위 사실** | "~가 어떤 방식으로 위반하였는가?" | `2. 사실의 인정 및 위법성 판단` |
| **처분/과징금** | "~에 대한 시정조치와 과징금은?" | `주문` + `3. 처분` |
| **법리 판단** | "~의 위법성 판단 근거는?" | `2. 위법성 판단` (법리 부분) |
| **복수 기업 비교** | "~와 ~가 벌인 위반 사례의 위법성을 대조하시오." | 각각 의결서의 `위법성 판단` |
| **일반 업종/도메인** | "~업종의 주요 불공정 행위 양상을 의결서 내용들을 바탕으로 분석하시오." | 각각 의결서의 `위법성 판단` |

### 질문 작성 시 주의사항

1. **대괄호 `[]` 사용 금지** — 검색 알고리즘 학습에 방해되지 않도록 제외
2. **자연스러운 질의 생성 (조사 및 단어 정제)** — `(주)`, `주식회사` 등 불필요한 회사명 접사를 제거하고, 앞 단어의 받침 유무를 판별하여 '은/는/이/가/와/과/을/를' 조사를 동적으로 붙여 자연스러운 문장을 생성
3. **분류 체계의 구체화 (Hybrid JSON 내용 반영)** — "공정거래(독과점/담합)", "갑을관계(불공정거래)" 등 괄호가 포함된 부자연스러운 위반분야는 실제 Hybrid JSON 텍스트에 포함된 키워드(담합, 시장지배, 하도급, 가맹 등)를 분석하여 "독과점", "담합", "갑을관계", "소비자보호" 등 사람이 실제 사용할 법한 단어로 변환
4. **질문의 일반화 (복수 의결서)** — 사용자는 특정 개수의 문서를 바탕으로 질문하지 않는 경향이 있으므로 "두 가지 의결서"와 같은 표현을 피하고, "의결서 내용들을 바탕으로" 등으로 일반화
5. **업종/도메인 중복 방지** — 여러 의결서가 동일 업종일 경우, "~업종과 ~업종"으로 중복되지 않게 단일 업종으로 통합 명시 (예: 제조/건설업 업종의 주요 불공정 행위 양상)
6. **검색 난이도를 다양하게** — 단순 요약부터 여러 의결서의 공통점/차이점을 분석하는 수준까지 다양화

---

## 3. 정답 chunk 선정 및 answer_reason 작성 원칙

### 선정 기준 (우선순위)

1. **위법성/처분/주문 청크 중심**: 가급적 위법성 판단 근거와 구체적인 행위 사실을 담은 `위법성 판단` 관련 text 청크를 우선 3개 추출.
2. 추가로 `주문` 및 `처분` 섹션 청크를 더해 5개 할당.
3. 복수 의결서의 경우 두 의결서에서 각각 3개, 2개 비율로 5개의 위법성 관련 청크 추출.

### answer_reason 작성 기준
단순히 의결서 제목을 반복하는 것이 아니라, `{기업명}의 {위반분야}에 대한 위반 사실 및 법리 판단 근거`와 같은 일관성 있는 형태로 작성하여, 각 청크가 왜 질문의 정답이 되는지 명확히 설명할 것. (상위 분류 카테고리: 갑을관계, 공정거래, 소비자보호 등에 맞춰 정리)

---

## 4. 의결서 구조 패턴 (Hybrid JSON 공통)

```
주문          → CH-001 (대부분 1개, 시정명령/과징금 요약)
이유
├─ 1. 기초사실       → 피심인 현황, 시장 구조, 거래 관계
├─ 2. 사실의 인정 및 위법성 판단  → 행위사실, 법리 판단 (가장 많은 청크)
├─ 3. 처분           → 시정조치, 과징금 산정 기준/금액
└─ 4. 결론           → 최종 의결 (대부분 1~3개)
```

- **chunk_type**: `text` 또는 `table`
- **chunk_id 형식**: `DOC-{uuid}-CH-{3자리 순번}`

---

## 5. 생성된 QA 샘플 요약

현재 총 600개의 데이터셋으로 구성되어 있습니다.
- 단일 의결서 QA 500개 (`Q-S-001` ~ `Q-S-500`)
- 복수 의결서 QA 100개 (`Q-M-001` ~ `Q-M-100`)
